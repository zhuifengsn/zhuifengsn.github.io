枚举类

枚举在曰常生活中很常见，例如一个人的性别只能是“男”或者“女”，一周的星期只能是 7 天中的一个等。类似这种当一个变量有几种固定可能的取值时，就可以将它定义为枚举类型。

枚举可以用来定义固定类别的实例。枚举（enum） 本质上就是一种特殊的类（class），但枚举类不能被继承，所有枚举类都继承自 java.lang.Enum 抽象类，并实现了 java.lang.Serializable 和 java.lang.Comparable 两个接口。

语法

简单版本

访问修饰符 enum 名称{
    成员1,成员2,成员3,...成员n;		// 以逗号分隔，以分号结束。（如果只有成员，没有添加方法，最后一个成员后可以不加符号，或以逗号结束也可以，不会报错）
}
// 任意两个枚举成员不能具有相同的名称
// 枚举成员是常量（系统会自动添加 public static final），不可被改变。
// 枚举成员实际上就是枚举类的实例，枚举类是不允许在类的外面去创建实例（new 一个实例）的，都是放到枚举类自己内部来创建实例。

最简单的例子：

public enum SimpleCategory {
    SPRING,
    SUMMER,
    autumn,
    Winter;
}

// 调用类
public class TestUse {
    public static void main(String[] args) {
        System.out.println(SimpleCategory.SPRING);      // SPRING
        System.out.println(SimpleCategory.SUMMER);      // SUMMER
        System.out.println(SimpleCategory.autumn);      // autumn
        System.out.println(SimpleCategory.Winter);      // Winter
        System.out.println(SimpleCategory.SPRING.name());       // SPRING
        System.out.println(SimpleCategory.SPRING.ordinal());    // 0
        System.out.println(SimpleCategory.autumn.name());       // autumn
        System.out.println(SimpleCategory.autumn.ordinal());    // 2
    }
}

枚举的元素没有所谓的数据类型，和 java 的变量命名一样，区分大小写，用的时候，你定义的是大写就得大写，小写就得小写（得一模一样，否则调用不到）。枚举元素按定义顺序会自动获得序号（ordinal），序号从 0 开始。

枚举可以定义成单个文件，也可以多个 enum （只能有一个 public enum）放在 .java 文件中。基本等同于 class 的定义模式，不加 public 的 enum 可以随便放，加了 public 需要放在同名文件中（文件名后缀为 .java）。

复杂版本

实际上枚举还可以定义构造方法、成员方法、成员变量等。

枚举的构造方法的访问修饰符必须为 private，可以不写访问修饰符（一般就是不写的），java 会帮忙自动加上 private。

这是因为，枚举的实例都是在枚举类的内部创建完成的，不需要也不能够在外部创建，所以访问修饰符只能是 private ，很合理。

构造方法与实例的关系
public enum ComplexCategory {
    SPRING,
    Summer(),				// 没参数的话，直接省略括号是一样的，都会调用无参的构造方法
    autumn(3),				// 有参数的话则调用有参的构造方法
    WINTER;
    ComplexCategory(int num){
        System.out.println("Arguments work. "+num);
    }
    ComplexCategory(){
        System.out.println("Simple work.");
    }
}

// 调用类
public class TestUse {
    public static void main(String[] args) {
        System.out.println("Test Enum...");					// Test Enum...
        System.out.println(ComplexCategory.autumn.name());	// autumn
        System.out.println(ComplexCategory.autumn);			// autumn
//        System.out.println(ComplexCategory.autumn());   // 报错，找不到 autumn() 方法
//        System.out.println(ComplexCategory.autumn(3));  // 报错，找不到 autumn(int) 方法
        System.out.println(ComplexCategory.SPRING);			// SPRING
    }
}

// 输出：
// Test Enum...
// Simple work.					----> 真的调用 enum 枚举类时才会创建枚举类中的枚举对象实例
// Simple work.
// Arguments work. 3
// Simple work.					----> 有多少个实例，就调用多少次构造方法，所以这里调用了 4 次。
// autumn
// autumn
// SPRING
枚举成员实例的创建时机

可以发现，当程序执行时，并不会马上去创建 enum 类中定义的实例，而是只在真的使用到这些实例时，才会去一次性调用相应的构造方法创建这个 enum 类中的所有实例，并且是依照实例在 enum 类中的顺序，依此创建实例。

枚举成员与构造方法

  • 枚举成员可以是没有参数的，此时可以直接省略括号，创建该枚举成员实例时，会调用无参数的构造方法；

  • 枚举成员有参数的，会调用有参数的构造方法。并且这个实参是在 enum 类中传入的（写死了，固定了），而非调用类中动态传入。

  • 枚举成员名称不能重复，即便用相同的名字分别定义一个有参的和无参的，也是非法的，会报错。

  • 枚举成员间以逗号分隔，最后以分号结束。

    如果只有成员，没有添加方法，最后一个成员后可以不加符号，或以逗号结束也可以，不会报错。

  • 枚举成员是常量（系统会自动添加 public static final），不可被改变。

  • 枚举成员一般用大写字母，但这不是语法强制要求。

public enum Season{
    SUMMER,
	SUMMER(3);		// 报错
    
    Season(){
        
    }
    Season(int num){
        
    }
}
枚举成员传入参数的作用

实际上不管这个枚举成员传入不传入参数，最后这个创建出来的实例都是这个名称。比如说 Spring(3) 与 SUMMER 两个成员实例，实际上使用时，都是 Season.Spring 与 Season.Summer 。用法上没有区别。你非得在调用类中加上括号 Season.SUMMER()，或在括号里进一步加入参数 Season.Spring(3)，都会报错。

因此，这个传入参数仅仅是在 enum 内部有用。用于调用一些方法，设置一些成员属性值等。

public enum Season {
    winter,
    Spring(3),
    SUMMER();

    private int number;
    Season(){
        System.out.println(this.number);
    }
    Season(int num){
        this.number = num;
        System.out.println(this.number);
    }

    public int getNumber(){
        return this.number;
    }
}

// 调用类
public class TestUse {
    public static void main(String[] args) {
        System.out.println(Season.Spring);                  // Spring
        System.out.println(Season.Spring.getNumber());      // 3
        System.out.println(Season.SUMMER.getNumber());      // 0
        System.out.println(Season.winter.getNumber());      // 0
    }
}
// 输出：
// 0				---> 开始调用构造方法创建实例
// 3
// 0				---> 依此创建好实例
// Spring
// 3
// 0
// 0

// 可以看到，这些传入的参数定义的成员属性值，的确是和当前实例捆绑的，是属于该实例的。
枚举成员的比较

由于枚举成员实例是在枚举类中创建好的，只有那么几个，所以指向其实例的引用相互比较时，可以直接使用 == （即直接比较是否指向同一个实例）。

常用方法

Java 中的每一个枚举都继承自 java.lang.Enum 类。当定义一个枚举类型时，每一个枚举类型成员都可以看作是 Enum 类的实例。当使用枚举类型成员时，直接使用枚举名称调用成员即可。

所有枚举实例都可以调用 Enum 类的方法。

枚举类名.values();								// 以数组形式返回枚举类型的所有成员
.ordinal();								// 获取枚举成员的索引（序号）位置（从 0 开始排序，顺序指的就是枚举类中实例定义的顺序）
.name();								// 返回枚举成员的名字

.compareTo();							// 比较两个枚举成员在定义时的顺序。若 序号2成员.compareTo(序号4成员)，则返回 2 - 4 = -2。（即返回前者序号减去后者序号）

枚举类名.valueOf("enum类中实际定义了的实例名");	 // 将普通字符串转换为枚举实例
// 之所以有这种获取枚举的实例的方式，而不采用 “枚举类.枚举成员名”，是因为有时候这个枚举成员名是由用户从键盘输入的。这提供了一种更动态的枚举实例获取方式。

public enum SimpleCategory {
    SPRING,
    SUMMER,
    autumn,
    Winter;

    public void sayHi() {
        System.out.println("Hi, " + this);
    }

    public void sayHi(String name) {
        System.out.println("Hi, " + this.name() + ". I'm " + name + ".");
    }

    @Override
    public String toString() {
        return "SimpleCategory{" +
                this.ordinal() +
                ":'" +
                this.name() + '\'' +
                '}';
    }
}

// 调用类
public class TestUse {
    public static void main(String[] args) {
        for(SimpleCategory a : SimpleCategory.values()){
            
            System.out.println(a);			
            // 实际调用的就是System.out.println(a.toString());
            // 如果没有在枚举类中覆盖 .toString() 方法，打印的就是枚举成员的名称。
            
            System.out.println(a.name());
            System.out.println(a.ordinal());
        }
        // System.out.println(SimpleCategory.valueOf("spring"));       // 报错。因为没有这个枚举成员实例
        System.out.println(SimpleCategory.valueOf("SPRING"));   // SimpleCategory{0:'SPRING'}
    }
}
// 输出：
// SimpleCategory{0:'SPRING'}
// SPRING
// 0
// SimpleCategory{1:'SUMMER'}
// SUMMER
// 1
// SimpleCategory{2:'autumn'}
// autumn
// 2
// SimpleCategory{3:'Winter'}
// Winter
// 3
// SimpleCategory{0:'SPRING'}
