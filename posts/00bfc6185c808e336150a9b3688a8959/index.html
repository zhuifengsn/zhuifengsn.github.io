linxu重启网络服务失败——Failed to start LSB: Bring up/down networking.

一、出现问题的场景

  1. 在虚拟机中的Linux系统启动后，发现没有网络，执行ifconfig 发现自己配置的ens33网卡没有启动。

  2. 接着，执行systemctl restart network 重启网络服务失败

  3. 查看网络状态，执行 systemctl status network

    发现报错了！

    Failed to start LSB: Bring up/down networking

二、排错

  1. 执行systemctl status NetworkManager，发现它开着
  2. 关掉它systemctl stop NetworkManager
  3. 重启网络服务 systemctl restart network
  4. 查看网络状态ifconfig 或者 systemctl status network，我配置的网卡ens33正常启动了！
  5. 关闭 NetworkManager 开机自启动 systemctl disable NetworkManager，避免下次开机又出现这个问题！

三、为什么出现这个问题？

NetworkManager服务和network.service服务有冲突，只能开启一个哦！
