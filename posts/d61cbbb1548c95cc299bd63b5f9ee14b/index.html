Linux httpd配置

1.使用域名访问
1）修改Linux中的 /etc/hosts文件(windows在)

[root@localhost /]# vim /etc/hosts
127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
192.168.10.100 www.rhce.com www.rhcsa.com #添加IP对应的域名

2）创建文件

[root@localhost /]# mkdir -p /www/name/{rhce,rhcsa}
[root@localhost /]# echo "this is rhce page" > /www/name/rhce/index.html
[root@localhost /]# echo "this is rhcsa page " > /www/name/rhcsa/index.html

3)在/etc/httpd/conf.d/目录下创建name.conf文件并进行配置

[root@localhost conf.d]# vim name.conf
<Directory "/www/name">
AllowOverride none
Require all granted
</Directory>

<VirtualHost 192.168.10.100:80>
DocumentRoot "/www/name/rhce"
ServerName www.rhce.com
</VirtualHost>

<VirtualHost 192.168.10.100:80>
DocumentRoot "/www/name/rhcsa"
ServerName www.rhcsa.com
</VirtualHost>

4）设置selinux和防火墙 并重启服务

[root@localhost /]# setenforce 0
[root@localhost /]# getenforce 
Permissive    #宽容模式
[root@localhost /]# firewall-cmd --add-port=80/tcp --permanent --zone=public 
[root@localhost /]# firewall-cmd --add-service=tcp --permanent --zone=public 
[root@localhost /]# firewall-cmd --reload      #重启防火墙，配置才能生效
[root@localhost /]# systemctl restart httpd    #重启httpd服务

5）结果如图

2.使用https访问: 使用ssl配置
1)创建文件

[root@localhost /]# mkdir -p /www/https
[root@localhost /]# echo "this is https" > /www/https/index.html

2)安装ssl

[root@localhost /]# yum install mod_ssl -y

3)在/etc/httpd/conf.d/目录下创建https.conf文件并进行配置

[root@localhost conf.d]# vim https.conf 
<Directory "/www">
AllowOverride none
Require all granted
</Directory>

<VirtualHost 192.168.10.100:443>
DocumentRoot "/www/https"
ServerName 192.168.10.100
SSLCipherSuite HIGH:MEDIUM:!aNULL:!MD5:!SEED:!IDEA
sslengine on
sslprotocol all -sslv2
sslcertificatefile /etc/pki/tls/certs/localhost.crt
sslcertificatekeyfile /etc/pki/tls/private/localhost.key
</VirtualHost>
~

4)设置selinux和防火墙 并重启服务(和上面一样，改端口就好)

5）结果,会显示不安全，因为是自己的证书，没有权威性

3.基于虚拟目录和用户控制：用户访问使用用户名和密码登录，使用alias实现虚拟目录
1）创建文件/www/mysecret/index.html

[root@localhost /]# mkdir -p /www/mysecret
[root@localhost /]# echo "this is mysecret" > /www/mysecret/index.html

2)创建用于登录的用户和密码

[root@localhost mysecret]# htpasswd -c /www/mysecret/passwords test
New password: 
Re-type new password: 
Adding password for user test

3）在/etc/httpd/conf.d/目录下创建vhost.conf文件并进行配置

[root@localhost conf.d]# vim vhost.conf 
<Directory "/www/mysecret">
AuthType Basic
AuthName "Hello"
AuthBasicProvider file
AuthUserFile "/www/mysecret/passwords"
Require user test
</Directory>

listen 8080
<VirtualHost 192.168.10.100:8080>
alias "/web" "/www/mysecret"
DocumentRoot "/www/mysecret"
ServerName  192.168.10.100
</VirtualHost>
~

4)设置selinux和防火墙 并重启服务(和上面一样，改端口就好)

5)结果如图

4.请求执行服务器上的cgi程序： scriptalias: 脚本目录映射， execcgi：增加目录执行权限， addhandler: 指定可以识别cgi程序后缀名
1)创建cgi程序文件

[root@localhost /]# mkdir -p /www/cgi
[root@localhost /]# vim /www/cgi/test.cgi
#!/bin/bash     #定义shell类型
printf "Content-Type: text/html;charset=utf-8\n\n"; 
printf "this is cgi\n";
~

2）在/etc/httpd/conf.d/目录下创建cgi.conf文件并进行配置

[root@localhost conf.d]# vim cgi.conf 
<Directory "/www/cgi">
AllowOverride none
Options +ExecCGI
AddHandler cgi-script .cgi .py
Require all granted
</Directory>

listen 2000
<VirtualHost 192.168.10.100:2000>
ScriptAlias "/cgi"  "/www/cgi"
ServerName 192.168.10.100
</VirtualHost>
~

4)设置selinux和防火墙 并重启服务(和上面一样，改端口就好)
