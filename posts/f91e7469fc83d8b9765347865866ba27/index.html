架构


上图即为使用noVNC技术的架构图，host0为novnc及代理websockify所在的宿主机，host1和host2即为要远程访问的客户机。

连接单台客户机

host1客户机

这里我们假定要连接的单台客户机为host1，系统为Ubuntu系统。

首先需要在host1上安装vnc服务器：

sudo apt-get install vnc4server

然后启动vncserver：

vncserver
或
vncserver :n

直接用vncserver启动时，第一次启动默认在5901端口，之后再启动每次默认加1
使用vcnserver :n启动时，启动的端口是5900+n

第一次启动时需要设置密码。

host0宿主机

首先安装git：

sudo apt-get install git

然后下载noVNC：

git clone https://github.com/novnc/noVNC.git

然后启动noVNC：

cd noVNC
./utils/launch.sh --vnc host1的IP:5901

第一次启动时如果没有安装websockify，会自动下载安装websockify

然后可以直接打开浏览器访问：
http://host0的IP:6080/vnc.html

连接多台客户机

客户机

需要在每一台客户机都安装vncserver，每个平台下可以是不同的vncserver，然后将vncserver启动。

host0宿主机

首先需要在websockify目录下新建一个配置文件token.conf，文件内容如下：

host1: host1的IP:host1的vncserver启动时的端口号
host2: host2的IP:host2的vncserver启动时的端口号

然后执行：

./utils/websockify/run --web=/home/ya/noVNC --target-config=./token.conf 6080

启动的./utils/websockify/run指的是websockify目录下的run指令，- -web指的是noVNC文件夹的根目录，- -target-config指的是token配置文件相对于websockify目录所在的位置，或者直接写在计算机中的绝对路径，6080是noVNC默认端口

然后访问：

http://host0的IP:6080/vnc.html?path=websockify/?token=host1

就可以访问到host1客户机
