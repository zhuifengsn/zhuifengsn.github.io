java开发 listener、servlet通过@Autowired取不到spring里的bean（bean值为null）

一、listener通过@Autowired取不到spring里的bean

1、问题描述

监听器里直接通过@Autowired注解取spring里的bean时是取不到的，

取到的都是null，调用bean方法就会报空指针异常。

2、原因分析

当调用bean内方法时，spring容器中还没有完成对注解bean的扫描，

spring-config.xml中配置的注解bean的优先级没有框架中的contextListener的优先级高，

也就是说，spring里bean的初始化完成是在contextListener之后的，

所以contextListener初始化的时候根据@Autowired扫描，肯定是null的。

3、解决方案

通过webApplicationContext取spring容器里的bean

以下是一个listener类方法的例子：

	// 程序启动时自动执行的监听器
	// 程序启动时动态获取数据库内的图书分类和所有图书，并将其添加到application域中，这样前端就可以在打开网页时加载到动态的图书信息
	public void contextInitialized(ServletContextEvent contextEvent) {
		ServletContext applaction = contextEvent.getServletContext();
		WebApplicationContext context = WebApplicationContextUtils.getWebApplicationContext(applaction);
		cService = (ICategoryService) context.getBean("categoryServiceImpl");
		iService = (IBookService) context.getBean("bookServiceImpl");
		List<Category> categories = cService.findAllCategories();
		List<Book> books = iService.findAllBooks();
		applaction.setAttribute("categories", categories);
		applaction.setAttribute("books", books);
	}

在这里需要注意一下bean的名称：spring容器默认将bean取名为首字母小写的类名

二、servlet通过@Autowired取不到spring里的bean

1、问题描述

servlet里直接通过@Autowired注解取spring里的bean时是取不到的，

取到的都是null，调用bean方法就会报空指针异常。

2、原因分析

在tomcat中，一个url请求进来到处理，经过这些步骤：

tomcat 启动，tomcat context加载（在这儿是spring context加载，因为spring的context实现了tomcat中容器接口），

filters,servletcontext启动。。。

在这儿servlet和applicationContext是两个平级的概念。

所以servlet也无法取到spring里的bean值

3、解决方案

1、改用其它mvc框架，如strus2或者springMVC。推荐springMVC，与spring无缝结合。

2、通过WebApplicationContext获取spring里的bean：

WebApplicationContext context = WebApplicationContextUtils.getRequiredWebApplicationContext(getServletContext());
 
running = context.getBean("beanName");
