学习Oracle Concepts-第6章 数据字典与动态性能视图

内容主要来源于Oracle官方文档(官网英文版和网络中文版)
Oracle Database Concepts

1.数据字典概述

数据字典是一组提供有关数据库管理元数据的只读表，包含如下信息：
1.数据库中模式对象的定义，包括列的默认值和完整性约束信息；
2.分配给模式对象和模式对象已经使用的空间量；
3.数据库用户的名称，授予用户的权限和角色、与用户相关的审计信息；
数据字典是数据库管理的核心部分，比如数据库会执行如下操作：
访问数据字典查找相关用户、模式对象很存储结构的信息；
每次发出DDL语言，修改数据字典；

1.1 数据字典的内容

数据字典由下列对象组成：
1、基表：这些基础表存储关于数据库的信息。只有数据库能够写和读这些表。用户很少直接访问基础表，因为它们已经被规范化，并且大部分数据被存储为隐藏格式；
2、视图：这些视图通过联接和过滤语句，简化基础表的数据称为有用的信息，例如用户或者表名。这些视图包含数据字典中所有对象的名称和描述。一些视图能够被所有数据用户访问，其他的职位管理员设计。

数据字典视图集
前缀用户访问内容注意
DBA_数据库管理员所有对象一些DBA_视图包含对管理员有用的额外列
ALL_所有用户用户有权限的对象包括用户拥有的对象。这些视图服从已启用的角色集合
USER_所有用户用户拥有的对象这些视图通常不包含OWNER列，被隐含为查询该视图的用户

不是所有视图都有这三个成员。例如有DBA_LOCK，没有ALL_LOCK。
系统提供了一个名为DICTIONARY的视图，包含所有数据字典视图的名称和简短的描述。

以DBA_开头的视图

显示全库的相关信息，DBA_开头的视图只是为数据库管理员设计的。
例如：查询数据库的所有对象

SELECT OWNER, OBJECT_NAME, OBJECT_TYPE
FROM DBA_OBJECTS
ORDER BY OWNER, OBJECT_NAME;
以ALL_开头的视图

以ALL_开头的视图从该用户的视角透视数据库。这些视图返回关于用户能访问的模式对象的信息，包括公共或显示时授予的权限和角色；以及用户拥有的模式对象。
例子：你能访问的所有对象

SELECT OWNER, OBJECT_NAME, OBJECT_TYPE
FROM ALL_OBJECTS
ORDER BY OWNER, OBJECT_NAME;

由于该视图遵从当前已启用的角色集，所以查询结果取决于启用了那些角色,默认情况下，角色是未启用的

SQL> SET ROLE ALL;
Role set.
SQL> SELECT COUNT(*) FROM ALL_OBJECTS;
COUNT(*)
----------
68295
SQL> SET ROLE NONE;
Role set.
SQL> SELECT COUNT(*) FROM ALL_OBJECTS;
COUNT(*)
----------
53771
以USER_开头的视图

最能引起普通用户兴趣的是以USER_开头的视图，这些视图：
1.指向用户在数据库的私有环境，包括用户创建的模式对象的元数据，用户的授权等等；
2.只显示用户相关行，返回ALL_视图的子集；
3.与其他视图有相同的列，除了隐藏起来的OWNER
4.为了方便可能会有PUBLIC同义词
例子：返回你的模式包含的所有对象

SELECT OBJECT_NAME, OBJECT_TYPE
FROM USER_OBJECTS
ORDER BY OBJECT_NAME;
DUAL表

DUAL 在数据字典中十一个小表，数据库和用户编写的程序可以引用她，保证一个已知的结果。当一个值必须返回一次是，DUAL表很有用。所有用户都可以访问DUAL表。

SQL> SELECT ((3*4)+5)/3 FROM DUAL;
((3*4)+5)/3
-----------
5.66666667
1.2 数据字典存储

数据字典基表是任何数据库创建是的第一批对象。所有数据库的数据字典表和视图都存储在system表空间中。由于数据库打开时，SYSTEM表空间始终处于联机状态，所以当数据库打开时，数据字典总是可用的。

1.3 Oracle如何使用数据字典

SYS用户拥有数据字典的所以基表和可访问视图。基表中的数据只能由数据库来写入或更改。

数据字典视图的公共同义词

数据库为许多数据字典视图创建了公共同义词，以便用户访问。用户应尽量避免使用与公共同义词相同的名称来命名它们自己的模式对象。

缓存数据字典为了快速访问

很多数据字典信息存在于数据字典缓存中，因为数据库需要这些信息来验证用户的访问权限和模式对象状态。

其他程序和数据字典

其他Oracle数据库产品可能会引用现有的视图，并创建自己的数据字典或视图。开发人员需要引用数据字典时应尽量引用公有同义词

2.动态性能视图概述

在数据库操作过程中，会维护一组记录当前数据库活动的虚拟表，这些视图被称为动态视图，因为它们在数据库处于打开状态和在使用过程中不断被更新。也成V$视图，包含如下所示信息：
系统和会话参数、内存使用和分配、文件状态、工作和任务进度、sql运行、统计和度量。

2.1 动态性能视图的内容

由于动态性能视图不能被数据库管理员更改或删除，有时也称为固定视图。但数据库管理员可以在表上执行查询和创建视图，并将这些视图的访问权限授予其他用户。
SYS拥有其名称以V_$开头的动态性能表。动态性能视图基于这些表创建，然后在视图上创建以V$作为前缀的公共同义词。
对于所有的V$视图，都有一个相应的GV$视图。在RAC中，查询GV$视图会从所有合适的数据库实例中检索其V$视图信息。
使用DBCA建库时，ORACLE会自动创建数据字典。会自动运行catalog.sql脚本，其中包含动态性能视图的视图和公共同义词的定义。

2.2 动态性能视图的存储

动态性能视图基于从数据库内存结构生成大的虚拟表。不是存储在数据库中的常规表，由于动态更新，不能保证读一致性。
由于不是真正的表，其数据决定于数据库和实例的状态。例如，当数据库start，没mount时（nomount），可以查询V$INSTANCE和V$BGPROCESS，但直到mount时（mount），才能查询V$DATAFILE。

3.数据库对象元数据

DBMS_METADATA软件包提供了提取数据库对象的完整定义的接口。定义可以按XML或SQLDDL来表示。
