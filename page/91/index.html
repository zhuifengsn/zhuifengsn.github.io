微信与此ipad不兼容电脑也显示设备版本过低9.0_苹果手机微信登录提示版本过低，请升级至最新版本的解决办法...

微信是我们现在日常生活中使用比较多的常用软件了，几乎每天我们都会使用到它，大家平时在使用微信的时候有没有发现一个问题，自己的手机或者ipad在恢复出厂设置或者刷机以后微信就用不了了，下载也下载不了，更新也没办法更新的情况？今天我们就来看看怎么解决。 今天刚好遇到一台ipad4平板电脑，客户描述电脑恢复出厂设置以后微信就一直登录不了，登录的时候提示版本过低，需要更新，但是点击去appstore里面更新又提示不兼容。 版本过低 更新的时候却又提示无法下载，这样的提示：很显然微信目前已经关闭了老版本微信的登录验证了，只有你安装了最新版本的微信才可以正常登录使用，但是ipad4的话比较老的产品了，最新系统就是10.3.3,很显然已经没有办法安装最新的微信了，难道就没有办法解决了么？ 经过反复研究，终于找到了解决办法，首先将你的ipad进行越狱，一些老版本的ipad设备都是可以直接越狱的，直接下载爱思助手一键越狱，然后打开cydia安装系统文件补丁。 我们越狱成功以后桌面上会生成一个cydia的软件，直接打开，找到软件源，点Cydia/Telesphoreo 点进去 找到箭头所示的插件，直接点击安装，安装成功以后ipad会注销。 注销开机以后重新连接电脑，打开爱思助手，如图所示爱思提示已越狱，说明越狱已经成功。 这个地方提示已越狱，如果这个地方提示有需要安装的插件，比如提示“安装AFC2” ， 那么就需要重新回到刚才安装afc2 的地方，重新安装下，直到上面这个字显示“已越狱”三个字，表示越狱成功。 之后微信怎么办呢？打开爱思助手，我的设备，会发现文件管理里，比之前多了很多东西，之前只有一个文件夹“文件系统(用户)”， 然后程序(用户) 里会找到微信。 接下来 打开"微信" 的文件夹，打开 WeChat.app ，找到info.plist配置文件，直接双击打开。 打开直接编辑 将这个版本，原来是7.0.2.24，改成 7.0.13.24， (13是最新版的)，这样保存就可以了。 然后重新打开微信，直接在ipad 上登录微信， 使用手机号密码登录即可登录成功。 版本会变成最新版7.0.13，然后就可以愉快的使用了。

（故障记录）关于多台服务器丢包现象

现象 发现几台服务器服务存在不稳定现象，ping测试发现这几台服务器对外都出现丢包现象，丢包率3%～5%，服务器为虚拟机部署在实体服务器上，一台实体机上部署数个虚拟机。 排查 一，通过arp确认出现问题的服务器都属于同一个实体机，继续从网络角度排查问题。 二，检查从路由层到服务端全链路状态，接口状态正常，网络设备收发光功率正常。 三，检查接口下丢包情况，发现接入层下联口的聚合组出现丢包，继续确认发现为固定一接口丢包，但该接口收发光正常。检查服务端收光，发现收光较低，更换光纤后恢复。 分析 链路少量丢包大部分由于链路情况较差导致，小部分是终端网卡问题。若丢包率近50%，可能是地址冲突。

java开发 listener、servlet通过@Autowired取不到spring里的bean（bean值为null）

一、listener通过@Autowired取不到spring里的bean 1、问题描述 监听器里直接通过@Autowired注解取spring里的bean时是取不到的， 取到的都是null，调用bean方法就会报空指针异常。 2、原因分析 当调用bean内方法时，spring容器中还没有完成对注解bean的扫描， spring-config.xml中配置的注解bean的优先级没有框架中的contextListener的优先级高， 也就是说，spring里bean的初始化完成是在contextListener之后的， 所以contextListener初始化的时候根据@Autowired扫描，肯定是null的。 3、解决方案 通过webApplicationContext取spring容器里的bean 以下是一个listener类方法的例子： // 程序启动时自动执行的监听器 // 程序启动时动态获取数据库内的图书分类和所有图书，并将其添加到application域中，这样前端就可以在打开网页时加载到动态的图书信息 public void contextInitialized(ServletContextEvent contextEvent) { ServletContext applaction = contextEvent.getServletContext(); WebApplicationContext context = WebApplicationContextUtils.getWebApplicationContext(applaction); cService = (ICategoryService) context.getBean("categoryServiceImpl"); iService = (IBookService) context.getBean("bookServiceImpl"); List<Category> categories = cService.findAllCategories(); List<Book> books = iService.findAllBooks(); applaction.setAttribute("categories", categories); applaction.setAttribute("books", books); } 在这里需要注意一下bean的名称：spring容器默认将bean取名为首字母小写的类名 参考自： 1、Spring消息监听器无法注入bean的问题 2、spring整合的项目中监听器Listener如何获取注入的bean（监听器获取Spring配置文件） 二、servlet通过@Autowired取不到spring里的bean 1、问题描述 servlet里直接通过@Autowired注解取spring里的bean时是取不到的， 取到的都是null，调用bean方法就会报空指针异常。 2、原因分析 在tomcat中，一个url请求进来到处理，经过这些步骤： tomcat 启动，tomcat context加载（在这儿是spring context加载，因为spring的context实现了tomcat中容器接口）， filters,servletcontext启动。。。 在这儿servlet和applicationContext是两个平级的概念。 所以servlet也无法取到spring里的bean值 3、解决方案 1、改用其它mvc框架，如strus2或者springMVC。推荐springMVC，与spring无缝结合。 2、通过WebApplicationContext获取spring里的bean： WebApplicationContext context = WebApplicationContextUtils.
